font-family Doesn’t Fall Back the Way You Think

Harry Roberts explains a subtle CSS rule: once an element declares its own font-family, fallback resolution happens inside that declaration instead of inheriting the parent stack. It is worth keeping because it turns a common assumption into a concrete rendering and performance rule for design systems.

  • font-family inheritance carries the full computed value, and child declarations do not borrow fallback fonts from ancestors.
  • A declaration like h1 { font-family: "Open Sans"; } can fall back to the user agent default while the web font is still loading.
  • Typography tokens should include their own fallback stacks instead of relying on body or root defaults.
  • Font-stack mistakes can show up as visible flashes of wrong text and layout shift, not only as stylistic inconsistencies.

How To Improve UX In Legacy Systems

Vitaly Friedman frames legacy UX work as a migration problem across workflows, dependencies, and user trust rather than a visual refresh. It stays because it lays out practical migration strategies and makes the rewrite-vs-increment decision explicit.

  • Legacy systems often encode critical business knowledge, so discovery should start with workflow and dependency mapping rather than redesign instincts.
  • One broken legacy step can define the perceived quality of the whole product, even when the surrounding UI is modern.
  • Migration strategy should be chosen deliberately: big-bang, incremental, parallel, incremental-parallel, or targeted legacy UI upgrades.
  • Stakeholder trust and heavy-user involvement are part of the delivery work because you are migrating habits and operations, not only screens.

Reproducing the AWS Outage Race Condition with a Model Checker

Using AWS’s own outage post-mortem as a starting point, this piece models a simplified DynamoDB DNS-management race in Spin/Promela and shows how the failure can emerge from a bad interleaving between concurrent enactors. It is worth keeping because it demonstrates a concrete way to turn an incident narrative into an executable concurrency model.

  • Post-mortem narratives can often be reduced into a smaller state model that is precise enough to expose the failure mechanism.
  • Model checking is useful for reasoning about rare interleavings that logs and intuition can miss.
  • Cleanup logic around versioned plans can be as dangerous as apply logic when multiple workers operate independently.
  • Formal methods become practical when used as focused debugging tools for high-risk concurrency behavior.
